# Routing Guide

## Default Trigger Cases

Use the research-reading lane when the user asks for:
- a breakdown of a paper, whitepaper, RFC, spec, technical standard, or dense technical article
- comparison of multiple papers or competing technical approaches
- extraction of claims, assumptions, methods, limitations, evidence quality, or open questions
- simplification into plain English, ELI5, or executive-summary form
- synthesis across a literature set or document bundle
- practical takeaways from jargon-heavy material

## Strong Route Triggers

Route by default if any of these are true:
- source material is longer than roughly 5 pages
- multiple documents need comparison
- the user asks for structured breakdown or critique
- the user asks for methodology or evidence assessment
- the source is highly technical, mathematical, legalistic, or domain-specific
- the user explicitly says to send it to research-reading

## Do Not Route When

Answer directly when the request is:
- a short factual clarification
- a quick definition
- a lightweight opinion not requiring deep reading
- a brief follow-up answerable from existing analysis in a few minutes

## Intake Checklist

When delegating, pass:
1. user's goal
2. source material or links/files
3. target depth: ELI5, practitioner, or expert
4. desired output format
5. urgency or deadline, if any
