# Output Format

Default to this structure unless the user or manager asks for something else.

1. Executive summary
   - 5-8 bullets

2. What problem this is solving

3. Core ideas / claims

4. How it works / argument map

5. Evidence / method quality
   - what kind of support is being used
   - how strong it is
   - where the weak points are

6. Key assumptions

7. Limitations / risks

8. Practical implications
   - what a builder, operator, investor, researcher, or decision-maker should take away

9. Jargon glossary

10. Best next questions

## Audience Modes

### ELI5
- use simple language
- avoid jargon unless defined immediately
- use analogies sparingly

### Practitioner
- default mode
- optimize for actionability and decision usefulness
- keep the explanation concrete

### Expert
- preserve nuance
- include methodological caveats
- call out unresolved edge cases and conceptual weaknesses

## Bad Patterns To Avoid

Do not:
- paraphrase the abstract and stop there
- repeat jargon without translating it
- hide uncertainty behind vague wording
- produce giant unstructured prose blocks
- treat speculative claims as established facts
- confuse elegant writing with strong evidence
